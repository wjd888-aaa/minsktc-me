# MinskTC 部署指南（从 MongoDB → D1）

## 已完成的代码变更

1. **数据库**: 从 MongoDB Data API 改为 Cloudflare D1 (SQLite)
2. **API 代码**: 所有 CRUD 操作改用 D1 SQL 绑定 (`context.env.DB`)
3. **前端**: `item._id` 改为 `item.id`（D1 使用自增整数主键）
4. **数据库 Schema**: `schema.sql` — items 表定义
5. **配置**: `wrangler.toml` — D1 binding 声明

## 前提条件（你的电脑）

1. **Node.js** 已安装（验证: `node -v`）
2. **Git** 已安装（验证: `git --version`）
3. 能访问 github.com 和 api.cloudflare.com

## 第一步：上传代码

```bash
cd Desktop\minsktc-me
git push origin main
```

## 第二步：创建 D1 数据库

```bash
npx wrangler d1 create minsktc-db
```

输出会显示类似：
```
✅ Successfully created DB 'minsktc-db' in region APAC
Created your database using D1's new storage backend.
[[d1_databases]]
binding = "DB"
database_name = "minsktc-db"
database_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
```

复制 `database_id` 的值。

## 第三步：更新 wrangler.toml

打开 `wrangler.toml` 第 5 行，把 `database_id = ""` 改为上一步得到的 ID：

```toml
database_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
```

提交并推送：

```bash
git commit -am "Add D1 database ID"
git push origin main
```

## 第四步：应用数据库表结构

```bash
npx wrangler d1 execute minsktc-db --file=schema.sql
```

## 第五步：在 Cloudflare Dashboard 添加 D1 绑定

1. 打开 https://dash.cloudflare.com
2. 进入 **Pages** → **minsktc-me** → **Settings** → **Functions**
3. 找到 **D1 database bindings** → **Add binding**
   - **Variable name**: `DB`
   - **D1 database**: 选择 `minsktc-db`
4. 保存

## 第六步：部署

GitHub 推送会自动触发 Cloudflare Pages 部署。等待几分钟后访问：

- https://minsktc-me.pages.dev
- https://minsktc.me（DNS 传播后）

## 验证 API

```bash
curl https://minsktc-me.pages.dev/api/categories
curl https://minsktc-me.pages.dev/api/items
```

## 快速部署（跳过 GitHub，直接 CLI）

如果你想跳过 GitHub 自动部署，直接用 CLI：

```bash
npm run build
npx wrangler pages deploy dist --project-name=minsktc-me
```

这会自动读取 `wrangler.toml` 中的 D1 绑定配置。

## 遇到的问题

如果 Cloudflare Dashboard 打不开（ERR_CONNECTION_RESET），可能原因：
- 网络环境（VPN/代理）问题
- DNS 传播未完成
- Cloudflare 临时服务问题

建议换一个网络环境（如手机热点）操作。
